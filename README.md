# VK Community Insights Dashboard

Мини-приложение для анализа сообществ ВКонтакте:
— загрузка постов через VK API
— расчет метрик вовлеченности
— визуализация данных (графики, таблицы)
— экспорт в CSV и JSON
— кэширование запросов

Проект реализован как fullstack-приложение:
Laravel (API) + Vue 3 (SPA)

## Архитектура

Backend:
- Выделенный сервисный слой для интеграций (VKService)
- Laravel REST API
- Сервисный слой (VKService) для работы с внешним API
- Кэширование (Redis / in-memory) для снижения нагрузки
- Обработка ошибок и логирование

Frontend:
- Vue 3 + Composition API + TypeScript
- Pinia для управления состоянием
- Axios для работы с API
- Разделение на компоненты (форма, таблицы, графики)

Общение:
Frontend ↔ Backend через REST API

## Выбор стека

Хотя в задании рекомендованы Node.js / Python, для реализации был выбран следующий стек:

Backend: Laravel (PHP)  
Frontend: Vue 3 (SPA)

### Причины выбора

**1. Скорость разработки и стабильность**
Laravel позволяет быстро реализовать REST API с понятной структурой, встроенной валидацией и удобной работой с HTTP-клиентом, что критично для задач с внешними API.

**2. Контроль над внешним API**
Задача предполагает работу с VK API (лимиты, ошибки, нестабильность), поэтому backend-слой был выбран осознанно:
— централизованная обработка ошибок  
— кэширование  
— логирование  
— защита токена  

**3. Разделение ответственности**
Frontend отвечает только за отображение и UX, backend — за бизнес-логику и интеграции.

**4. Vue 3 как легковесный SPA-фреймворк**
Vue выбран как быстрый и гибкий инструмент для:
— построения интерактивного дашборда  
— работы с состоянием (Pinia)  
— интеграции с API  

**5. Практичность**
Стек позволяет быстрее довести решение до рабочего состояния без потери качества, что важно в условиях ограниченного времени.


## Работа с VK API

Используется метод wall.get для получения постов.

Особенности:
- реализована пагинация
- обработка rate limit (429)
- защита от падений API
- нормализация данных перед передачей на frontend

## Кэширование

Результаты анализа кэшируются на 10–30 минут.

Зачем:
- уменьшение количества запросов к VK API
- ускорение повторных запросов
- защита от rate limits

Ключ кэша:
group_id + период

## Возможности

- Анализ постов за период
- Топ постов по вовлеченности
- Средняя вовлеченность
- Распределение типов контента
- Динамика активности по дням
- Экспорт данных (CSV, JSON)

## Удобство разработки

Makefile используется как удобная обертка над Docker Compose

Примеры:

make dev-up      # сборка и запуск контейнеров  
make dev-down    # остановка  
make dev-serve   # запуск frontend dev-сервера

## Принятые решения и компромиссы

- Backend используется для кэширования и обработки API (вместо прямых запросов с frontend)
- Выбран Laravel как быстрый способ построения API
- Использован SPA (Vue) для интерактивного UI
- Кэш реализован на уровне backend, а не frontend, чтобы избежать дублирования логики

## Особенности реализации

- защита от rate limiting VK API
- нормализация и агрегация данных на backend
- минимизация количества запросов за счет кэширования
